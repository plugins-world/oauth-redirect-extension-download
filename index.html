OAuth 地址重定向助手

自动重定向 OAuth 回调地址，支持配置多条域名映射规则，解决网络隔离环境下的授权问题

下载 v1.1.3 GitHub

配置页面预览

核心特性

多规则配置

支持配置多个域名重定向规则，满足不同场景需求

可视化界面

直观的配置界面，无需手动编辑代码，轻松管理规则

实时生效

配置保存后立即生效，无需重启浏览器

参数完整性

保持 URL 参数完整性，只替换域名部分

隐私安全

所有数据本地存储，不收集任何用户信息

灵活控制

可单独启用/禁用每条规则，灵活应对不同场景

安装方法

1

下载扩展

点击上方 "下载扩展" 按钮，下载最新版本的 zip 文件

2

解压文件

将下载的 zip 文件解压到本地目录

3

打开扩展管理页面

在 Chrome 浏览器中访问 chrome://extensions/

4

开启开发者模式

点击页面右上角的 "开发者模式" 开关

5

加载扩展

点击 "加载已解压的扩展程序"，选择解压后的目录

开始使用

点击浏览器工具栏的扩展图标，开始配置重定向规则

使用场景

🏢

网络隔离环境

内外网隔离，OAuth 服务只能配置内网回调地址，但开发者在外网无法访问

💻

开发环境切换

需要在不同环境（开发/测试/生产）之间快速切换 API 地址

🔐

第三方服务回调

第三方服务只允许配置一个回调地址，需要重定向到本地开发环境
